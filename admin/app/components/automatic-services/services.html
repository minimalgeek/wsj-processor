Automatic Services

Stooq - Download missing forex data

Every 1 hour, at HH:00

Downloads the latest forex data from http://stooq.com, and saves it into MySQL forex table

Yahoo - Download tick forex data

Every 5 seconds - at the moment it's turned off!

Downloads the latest forex data from Yahoo Finance, and saves it into MySQL forex table

Spice Indices - Download S&P members

Every day, at 00:00

Checks for changes in the S&P 100, 400, 500, 600 lists on https://www.spice-indices.com, and saves them to z:\Quant_CO2\SAndP\
The US.tls file is very important, because it is used in several services as the main ticker list

SeekingAlpha and Zacks integration

Every day, at 00:00

When we have a future report date on http://zacks.com, it schedules 4 jobs for the following 4 days, to check arriving Earnings Call Transcripts on http://seekingalpha.com/
We save it to earnings_call collection, and the report dates are stored previously in zacks_earnings_call_dates

Zacks - earnings call dates downloader, 2nd implementation

Every day, at 06:00

Based on the US.tls ticker file, it downloads all the earnings call dates and saves them to zacks_earnings_call_dates_ii mongo collection

The method is the same, like in NEW - Refresh all report dates from http://zacks.com function

MongoDB - Create flat collections

Every day, at 08:00

We check the JavaScript (xyz.js) files of z:\Quant_CO2\mongo_scripts\, and run them on MongoDB, insider database
It is necessary for the AmiBroker - MongoDB Bridge, because AmiBroker can not work on complex objects

WSJ - collect new article contents

Every 30 minutes, at HH:30 and HH:00 - at the moment it's turned off!

It opens the archive site to check for new articles on http://wsj.com, then saves them to wsj database, article collection
